Sobre o Projeto

Esta aplicação web demonstra o uso de HTML semântico, CSS para estilização e layout, e JavaScript para interatividade, validação, armazenamento local e consumo de dados de uma API externa.

Conceitos Utilizados:

  • HTML Semântico: Utilização de tags como <header>, <nav>, <main>, <section>, <article>, <form>, <table>, etc., para estruturar o conteúdo de forma clara e significativa.
  • CSS:
    • Box Model: Compreensão de como as propriedades de margem, borda, preenchimento e conteúdo afetam o tamanho e o layout dos elementos.
    • Flexbox: Utilizado na tela de cadastro para organizar os elementos do formulário de forma flexível e responsiva.
    • Grid: Utilizado na tela de lista de usuários para criar um layout de tabela responsivo e organizado.
    • Media Queries: Implementadas para adaptar o layout da lista de usuários a diferentes tamanhos de tela, garantindo a responsividade.
    • Tipografia e Hierarquia Visual: Estilização do texto para melhorar a legibilidade e destacar informações importantes.
    • Layout Fluido: Utilização de unidades relativas para criar layouts que se adaptam ao tamanho da tela.
  • JavaScript:
    • Interações no DOM: Manipulação dos elementos HTML para criar funcionalidades dinâmicas.
    • Validações em Tempo Real: Verificação dos dados do formulário de cadastro enquanto o usuário digita.
    • Armazenamento Local (LocalStorage): Utilização do localStorage para salvar e recuperar os dados dos usuários no navegador.
    • Fetch API: Realização de requisições HTTP para consumir dados da API de consulta de CEP.
    • Manipulação de Promises e Respostas HTTP: Tratamento das respostas assíncronas da API.
  • Fetch API: Interface para buscar recursos de forma assíncrona na web.
  • LocalStorage: Mecanismo de armazenamento web que permite salvar dados localmente no navegador do usuário.
